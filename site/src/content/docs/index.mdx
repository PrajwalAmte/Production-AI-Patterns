## The 8 Pillars

<CardGrid>
  <Card title="Inference & Serving" icon="rocket">
    Gateway routing, semantic caching, model routers, fallback chains, and batch
    inference patterns for cost and latency control.
  </Card>

  <Card title="Data Patterns for AI" icon="document">
    Data contracts, feature stores, training pipelines, eval dataset management,
    and streaming data for real-time AI systems.
  </Card>

  <Card title="Reliability & Resilience" icon="approve-check-circle">
    Circuit breakers, graceful degradation, canary deployments, shadow testing,
    and model rollback for AI-specific failure modes.
  </Card>

  <Card title="Retrieval & Memory" icon="magnifier">
    Classic RAG, GraphRAG, hybrid search, reranking, contextual memory, and
    chunking strategies for knowledge-grounded systems.
  </Card>

  <Card title="Observability & Monitoring" icon="seti:todo">
    Span-level tracing, cost attribution, quality drift detection, prompt
    logging, and SLOs designed for AI workloads.
  </Card>

  <Card title="Security & Trust" icon="warning">
    Input sanitization, output validation, structured enforcement, audit trails,
    least-privilege tool access, and PII scrubbing.
  </Card>

  <Card title="Cost & Efficiency" icon="setting">
    Token budgets, tiered model strategies, prompt compression, inference-time
    compute scaling, and cost circuit breakers.
  </Card>

  <Card title="Governance & Compliance" icon="list-format">
    Model cards, data lineage, policy-as-code, responsible AI checklists,
    human-in-the-loop gates, and model versioning.
  </Card>
</CardGrid>
